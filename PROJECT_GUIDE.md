# 🚀 WorkSpace Management System - Project Guide

A full-stack employee and task management system featuring a dual-mode authentication flow (Admin/OTP and Employee/Login Code), role-based access control, and a modern glassmorphism UI.

---

## 🛠️ Tech Stack & Rationale

### Backend (FastAPI + PostgreSQL)
- **FastAPI**: Used for its high performance (Python's fastest), automatic interactive documentation (Swagger), and native asynchronous support.
- **SQLAlchemy (ORM)**: Provides a powerful and flexible way to interact with PostgreSQL using Python objects, making database operations safer and more readable.
- **PostgreSQL**: A robust, industrial-grade relational database used for data integrity, complex queries, and reliable storage.
- **Pydantic**: Used for data validation and serialization, ensuring that API inputs and outputs always match the expected format.
- **JWT (Jose)**: Implements stateless authentication, allowing the server to verify user identity and roles without keeping a session database.

### Frontend (Next.js + Vanilla CSS)
- **Next.js 14 (App Router)**: The leading React framework, chosen for its excellent developer experience, optimized performance, and easy routing.
- **TypeScript**: Adds static typing to JavaScript, catching errors during development and improving code maintainability.
- **Vanilla CSS**: Used for maximum control over the "Premium Glassmorphism" design without the overhead of heavy CSS frameworks.
- **Axios**: A clean, promise-based HTTP client used to communicate with the FastAPI backend.
- **Lucide React**: A beautiful, consistent icon set that complements the modern UI design.

## 🔐 Authentication & Permissions Flow

The system uses a unique **Identity-First** login flow:

1. **Phone Verification**: The user enters their phone number.
2. **Role Detection**: The backend checks if the number belongs to an **Admin** or an **Employee**.
3. **Dual Verification**:
   - **Admins**: Receive a 6-digit **OTP** (simulated in console) for secure access.
   - **Employees**: Enter their unique **Login Code** (e.g., `EmpUDT2`) generated by the admin.
4. **Permissions**:
   - **Admin**: Can create/edit/delete employees and tasks.
   - **Employee**: Can only see tasks assigned to them and update their own task `status`.

## 🚀 Getting Started

### 1. Run the Backend
```bash
cd backend
python -m venv venv
source venv/Scripts/activate  # Windows
pip install -r app/requirements.txt
uvicorn app.main:app --reload
```

### 2. Run the Frontend
```bash
cd frontend
npm install
npm run dev
```
